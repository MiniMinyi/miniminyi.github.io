MyFirstIOSapp

6月 09, 2016 发布在 blog

写了一个获取传感器数据并用TCPsocket发送的ios程序，在服务器端用python的matplotlib库来画图，
用库twisted来实现管理client，接受信息的功能。

ios 端

界面设计

如果页面设计比较简单，只考虑在一种设备上运用的话，只需要在Main.storyboard中把各种摆件摆好，然后全选，选右下角有三角形的按键，然后选择Reset To Suggested constrains。
但如果要适应于其他设备，而且想要在横屏竖屏的显示都比较正常，那么则需要添加Alignment Constrains 或者 Pin Constrains, 添加方法同样是选择要添加的控件，然后点右下角的按钮。

界面设计的比较好的教程是Kevin写的《给女朋友的IOS教程》中的AutoLayout BILIBILI链接 Youtube链接 在Youtube中观看的体验明显要好很多啊（因为能够加速 >_<）

界面与代码的结合

在代码中声明一个与界面控件相关的变量最方便的方法就是（在Xcode中）打开AssitantEditor，按住Ctrl，将控件拖动到代码中合适的位置。
要添加一个与界面控件相关的动作，也是一样的做法。

create TCPsocket

使用了别人写的一个库 SwiftSocket .
用法很简单，只要把.c和.swift文件加入自己的工程里，然后import Foundation之后就可以按照readme.md里的API使用了。
处理数据参考了一篇blog文Swift - 使用socket进行通信（附聊天室样例）
其中将格式处理为JSON格式和dispatch_async的用法都对我比较有帮助。

python实现的server端

twisted

在socket方面使用了twisted
然后跟着这篇教程Networking Tutorial for iOS: How To Create A Socket Based iPhone App and Server 中的Python部分，就可以写好了，教程解释得很详细，但是后面关于iOS方面是用C#写的，所以放弃了参考它。

matplotlib

使用了库 Matplotlib
有一篇很美观、简单、详细的中文教程Matplotlib教程，但是内容比较少
当然，最详细的就是官方的教程了，官网上还有很多demogallery，在学习前先在gallery中找与自己心目中相似的图，然后照着示例代码写，效率会比较高。

总结

  1. 总的来时，ios端和python端都只是实现了预想中最基本的功能，还有很多bug和做得不好的地方。
  2. 在这次编程中，初步学习了swift的语法知识，发现swift是很容易上手的语言，只是自己目前对其中很多语法不太了解，之后有时间一定要好好学习一下。
  3. Kevin写的给女朋友的ios教程勾起了我做App的兴致，可惜暑假时间不多，先留着这个小火种吧，以后再实现。
  4. Python有那么多库真的太赞了 ~\(≧▽≦)/~
